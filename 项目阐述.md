# 课程项目汇报：「你这朋友算是废了」交互式心理测评应用

## 一、项目背景与目标

### 1.1 项目概述

本项目是一个基于 Web 技术实现的交互式心理测评应用，采用讽刺幽默的风格探讨当代年轻人的精神状态与行为模式。用户通过完成 15 道情境选择题，系统根据用户的选择计算六个维度的属性分数，最终生成一个具有个性化特征的"画像标签"以及相应的评语反馈。

项目的核心设计理念在于通过轻松诙谐的文案风格，使用户在娱乐性的测试过程中获得情感共鸣，同时系统性地收集用户选择数据，通过算法逻辑生成具有参考价值的结果输出。整体视觉设计采用复古报纸风格，营造一种"严肃又荒诞"的氛围，与内容主题形成呼应。

### 1.2 项目目标

本项目的开发目标包含以下几个层面：

**功能性目标：**
- 实现完整的测评流程，包括开始页、答题页、结果页三个核心模块
- 构建基于权重的多维度计分系统，确保结果判定的准确性
- 设计卡牌收集系统，增加用户粘性与互动趣味性
- 支持结果分享功能，包括文案复制与图片保存

**技术性目标：**
- 采用纯前端技术栈，无需后端服务或构建工具，降低部署门槛
- 实现数据驱动的设计架构，核心内容通过 JSON 配置文件管理
- 确保响应式布局，适配桌面端与移动端设备

**体验性目标：**
- 设计流畅的页面切换动画与交互反馈
- 提供可回退修改答案的功能，增强用户控制感
- 通过进度指示器明确用户当前位置

## 三、核心机制详解

### 3.1 六维度属性系统

项目设计了六个核心属性维度，用于量化用户在测试中的行为倾向。每个维度对应特定的心理行为特征：

| 维度名称 | 代码标识 | 定义说明 | 典型表现 |
|---------|---------|---------|---------|
| 摆烂力 | `lan` | 放弃的速度与优雅程度（防御/闪避） | 逃避任务、拖延症、主动躺平 |
| 自嘲力 | `zic` | 把悲剧讲成喜剧的能力（精神攻击） | 自我调侃、黑色幽默、悲剧喜剧化 |
| 画饼力 | `hua` | 给自己立flag并相信的能力（魅惑/自我催眠） | 制定不切实际的计划、盲目乐观 |
| 消费力 | `xia` | 用花钱代替行动的执行力（资源置换） | 通过购物缓解焦虑、仪式性消费 |
| 表演力 | `bia` | 线上线下人格切换的熟练度（伪装/潜伏） | 社交面具、人设经营、内外不一致 |
| 钝感力 | `dun` | 对失败麻木并继续的能力（韧性/回血） | 情感钝化、习得性无助、情绪压抑 |

**权重设计原则：**

每道题目包含四个选项（A/B/C/D），每个选项对六个维度分配不同的权重值（0-5分）。权重设计遵循以下原则：

1. **单维度主导原则**：每个选项通常在1-2个维度上具有明显高分（3-5分），避免选项过于平均
2. **语义一致性原则**：选项文案与权重分配需保持逻辑一致
3. **平衡性原则**：同一题目的四个选项在总分上保持相近，避免选项间失衡

**权重示例（第4题：早起flag）：**

| 选项 | 摆烂 | 自嘲 | 画饼 | 消费 | 表演 | 钝感 | 总分 |
|-----|-----|-----|-----|-----|-----|-----|-----|
| A | 4 | 1 | 0 | 0 | 1 | 2 | 8 |
| B | 2 | 3 | 2 | 0 | 2 | 0 | 9 |
| C | 1 | 4 | 0 | 0 | 1 | 1 | 7 |
| D | 5 | 2 | 0 | 0 | 0 | 1 | 8 |

### 3.2 结果判定逻辑

结果判定是本项目的核心算法模块，采用三层优先级机制确保结果的准确性与合理性。

**判定流程：**

```
输入：用户六维度得分 totals
输出：匹配的结果标签 outcome

Step 1: 遍历所有结果标签，筛选满足条件者
Step 2: 按优先级分组（extreme > combo > core）
Step 3: 同优先级内按触发分数排序
Step 4: 返回最高优先级、最高分数的结果
Step 5: 若无匹配，执行兜底逻辑
```

**三层优先级详解：**

**第一层：极端标签（extreme tier）**
- 触发条件：单维度分数 ≥ 50
- 标签数量：6个
- 设计意图：奖励"极端"用户，提供戏剧性结果

示例：
```json
{
  "id": "extreme-lan",
  "tier": "extreme",
  "label": "摆烂终身荣誉顾问",
  "description": "放弃已成为肌肉记忆，建议申遗",
  "conditions": {
    "min": { "lan": 50 }
  }
}
```

**第二层：组合标签（combo tier）**
- 触发条件：三个指定维度分数均 ≥ 20
- 标签数量：5个
- 设计意图：识别复杂行为模式，增加结果多样性

示例：
```json
{
  "id": "combo-01",
  "tier": "combo",
  "label": "创伤驱动的赛博行为艺术",
  "description": "用放弃表演治愈童年，线上比线下更真实",
  "conditions": {
    "min": { "lan": 20, "zic": 20, "bia": 20 }
  }
}
```

**第三层：核心标签（core tier）**
- 触发条件：主维度 ≥ 25 且 副维度 ≥ 15
- 标签数量：15个
- 设计意图：覆盖主流用户群体，提供细分结果

示例：
```json
{
  "id": "core-01",
  "tier": "core",
  "label": "创伤驱动的行为艺术",
  "description": "用放弃表演治愈童年，观众只有自己。",
  "main": "lan",
  "sub": "zic",
  "conditions": {
    "min": { "lan": 25, "zic": 15 }
  }
}
```

**兜底机制：**

当用户得分无法触发任何预设条件时，系统执行兜底逻辑：

```javascript
function fallbackOutcome(scores) {
  // 1. 按分数降序排列所有维度
  const ordered = ATTRS.map(attr => ({ id: attr.id, value: scores[attr.id] }))
    .sort((a, b) => b.value - a.value);
  
  // 2. 取最高分维度作为main，次高分作为sub
  const topOne = ordered[0]?.id;
  const topTwo = ordered[1]?.id;
  
  // 3. 尝试匹配 core 标签
  let matched = coreOutcomes.find(o => o.main === topOne && o.sub === topTwo);
  if (!matched) {
    matched = coreOutcomes.find(o => o.main === topOne);
  }
  
  return matched || coreOutcomes[0];
}
```

### 3.3 卡牌系统

卡牌系统是项目的特色功能模块，为用户提供了额外的互动体验与收集乐趣。

**卡牌数据结构：**

```javascript
{
  id: "T1-A",                    // 唯一标识
  questionId: 1,                 // 对应题目ID
  optionId: "A",                 // 对应选项ID
  title: "后来这技能在KTV派上了用场",  // 卡牌标题（取自选项note）
  image: "./images/cards/t1a.png"      // 卡牌图片路径
}
```

**卡牌状态管理：**

系统维护三个独立的状态变量，管理卡牌在不同阶段的表现：

| 状态变量 | 类型 | 用途 | 更新时机 |
|---------|------|------|---------|
| `collectedCards` | Set | 记录已解锁卡牌ID | 用户选择选项时 |
| `drawnCards` | Array | 记录已抽取卡牌对象 | 进入结果页时 |
| `revealedSlots` | Array[3] | 记录三个槽位的翻开状态 | 用户点击卡牌时 |

**卡牌解锁流程：**

```javascript
function selectOption(question, option) {
  // 1. 记录用户选择
  state.answerMap[question.id] = { optionId: option.id, weights: option.weights };
  
  // 2. 解锁对应卡牌
  const cardId = getCardId(question.id, option.id);  // 如 "T1-A"
  state.collectedCards.add(cardId);
  
  // 3. 更新UI
  renderQuestion();
}
```

**卡牌抽取流程：**

```javascript
function initDrawCardsSection() {
  // 1. 从已解锁卡牌中随机抽取3张
  const collectedArray = Array.from(state.collectedCards);
  const shuffled = collectedArray.sort(() => Math.random() - 0.5);
  const drawnIds = shuffled.slice(0, 3);
  
  // 2. 存储抽取结果
  state.drawnCards = drawnIds.map(id => getCardById(id)).filter(Boolean);
  
  // 3. 重置翻开状态
  state.revealedSlots = [false, false, false];
  state.hasDrawn = false;
}
```

**回声拼句功能：**

当用户翻开全部3张卡牌后，系统将三张卡牌的标题拼接成一句话，展示在结果页：

```javascript
function composeCardSentence(cards) {
  // 1. 清理每张卡牌标题的结尾标点
  const cleaned = cards.map(card => normalizeSentenceFragment(card.title));
  
  // 2. 拼接成句
  return `${cleaned[0]}，${cleaned[1]}，${cleaned[2]}。`;
}

function normalizeSentenceFragment(text) {
  // 移除结尾的句号、感叹号、问号等
  return text.replace(/[。！？；,.，!？;]+$/g, "");
}
```

### 7.1 数据结构转换

**问题描述：**

- 设计六维度属性系统（摆烂/自嘲/画饼/消费/表演/钝感）
- 实现单次测评的结果判定逻辑
- 保持讽刺幽默的文案风格

**解决方案：**

1. **属性映射**：将原有属性重新归类到新维度
   - "躺平值" → "摆烂力(lan)"
   - "自嘲值" → "自嘲力(zic)"
   - "努力值" → 重新分配到"画饼力(hua)"
   - 新增"消费力(xia)"、"表演力(bia)"、"钝感力(dun)"

2. **题目重新设计**：保留人生阶段场景，改为独立题目

   - 共计26个结果标签

### 7.2 结果判定优先级处理

**问题描述：**

用户答题后，其六维度得分可能同时满足多个结果标签的触发条件。例如，用户可能在"摆烂"维度获得50分以上，同时满足多个组合标签的条件。如何确定最终显示的结果？

**问题分析：**

假设用户得分如下：
```
lan: 52, zic: 30, hua: 15, xia: 25, bia: 28, dun: 10
```

可能触发以下结果：
- `extreme-lan`（摆烂 ≥ 50）
- `combo-01`（lan ≥ 20, zic ≥ 20, bia ≥ 20）
- `core-01`（lan ≥ 25, zic ≥ 15）

**解决方案：**

引入优先级机制，按 `tier` 字段区分结果层级：

```javascript
function outcomePriority(outcome) {
  const priorityMap = { extreme: 3, combo: 2, core: 1 };
  return priorityMap[outcome.tier] ?? 0;
}

function selectOutcome(scores) {
  // 1. 筛选所有满足条件的结果
  const triggered = state.outcomes
    .filter(outcome => conditionsMet(outcome.conditions, scores))
    .map(outcome => ({
      ...outcome,
      triggerScore: computeTriggerScore(outcome, scores)
    }));

  // 2. 按优先级分组
  const maxPriority = Math.max(...triggered.map(o => outcomePriority(o)));
  const prioritized = triggered.filter(o => outcomePriority(o) === maxPriority);

  // 3. 同优先级内按触发分数排序
  const maxScore = Math.max(...prioritized.map(o => o.triggerScore));
  const top = prioritized.filter(o => o.triggerScore === maxScore);

  // 4. 如有多个，随机选择
  return top[Math.floor(Math.random() * top.length)];
}
```

**触发分数计算：**

触发分数用于在同优先级结果中进行排序，计算方式为所有触发维度的分数之和：

```javascript
function computeTriggerScore(outcome, scores) {
  const keys = Object.keys(outcome.conditions?.min || {});
  return keys.reduce((sum, key) => sum + scores[key], 0);
}
```

### 7.3 雷达图绘制与标签布局

**问题描述：**

需要使用 Canvas 绘制六边形雷达图，展示用户的六维度得分。但在实际绘制中，六个维度的标签容易与雷达图重叠，影响可读性。

**问题分析：**

六边形雷达图的六个顶点分布在 360° 圆周上，每个顶点间隔 60°。标签放置在顶点外侧，但在某些角度（如顶部和底部）标签容易与图形重叠。

**解决方案：**

1. **动态计算标签位置**

```javascript
ATTRS.forEach((attr, index) => {
  const angle = (Math.PI * 2 * index) / ATTRS.length - Math.PI / 2;
  const x = center + Math.cos(angle) * labelDistance;
  const y = center + Math.sin(angle) * labelDistance;
  
  // 根据角度调整文本对齐方式
  ctx.textAlign = Math.cos(angle) > 0.1 ? "left" 
                : Math.cos(angle) < -0.1 ? "right" 
                : "center";
  ctx.textBaseline = Math.sin(angle) > 0.1 ? "top" 
                   : Math.sin(angle) < -0.1 ? "bottom" 
                   : "middle";
  
  ctx.fillText(attr.label, x, y);
});
```

2. **设置动态留白**

```javascript
const textPadding = Math.max(26, Math.round(size * 0.14));
const labelOffset = Math.max(10, Math.round(size * 0.05));
const radius = Math.max(32, center - textPadding - labelOffset);
```

3. **高DPI适配**

```javascript
const ratio = window.devicePixelRatio || 1;
canvas.width = size * ratio;
canvas.height = size * ratio;
canvas.style.width = `${size}px`;
canvas.style.height = `${size}px`;
ctx.setTransform(ratio, 0, 0, ratio, 0, 0);
```

### 7.4 卡牌状态同步问题

**问题描述：**

卡牌系统涉及三个不同的状态阶段：解锁（答题时）、抽取（结果页）、展示（卡牌墙）。如何确保三个状态正确同步？

**问题场景：**

1. 用户选择选项后，卡牌应该被"解锁"
2. 用户重新答题时，之前解锁的卡牌应该保留还是清除？
3. 用户在结果页抽取卡牌后，卡牌墙应该显示抽取的卡牌

**解决方案：**

设计三个独立的状态变量，明确各自的职责：

```javascript
const state = {
  // 卡牌解锁状态（答题时更新）
  collectedCards: new Set(),      // Set<string> 存储已解锁卡牌ID
  
  // 卡牌抽取状态（结果页更新）
  drawnCards: [],                 // Array<Card> 存储抽取的卡牌对象
  revealedSlots: [false, false, false],  // Array<boolean> 三个槽位的翻开状态
  hasDrawn: false,                // boolean 是否完成抽卡
  
  // 状态重置逻辑
  function startQuiz() {
    state.collectedCards = new Set();  // 重新答题时清空解锁
    state.drawnCards = [];              // 清空抽取
    state.revealedSlots = [false, false, false];
    state.hasDrawn = false;
  }
};
```

**状态转换流程：**

```
答题选择 → collectedCards.add(cardId)
         ↓
完成答题 → 从 collectedCards 中随机抽取3张 → drawnCards
         ↓
点击卡牌 → revealedSlots[index] = true
         ↓
全部翻开 → hasDrawn = true → 生成回声拼句
```

### 8.4 调试过程中的问题记录

**问题1：选项旁白不显示**

- **现象**：选中选项后，旁白文字（note字段）不显示
- **原因**：CSS中设置了 `opacity: 0; visibility: hidden;`
- **解决**：通过 `.option.selected .option-content p` 选择器，在选中状态下设置 `opacity: 1; visibility: visible;`

**问题2：雷达图在高分屏模糊**

- **现象**：高分屏（如Mac Retina）上雷达图显示模糊
- **原因**：未考虑 `devicePixelRatio`
- **解决**：根据 `devicePixelRatio` 调整 Canvas 分辨率

**问题3：卡牌图片加载失败**

- **现象**：部分卡牌图片显示为破损图标
- **原因**：图片命名大小写不一致
- **解决**：统一命名规则为小写 `t{题号}{选项}.png`
