# 项目概述
- 单页测评应用：15 题 / 6 维度，结果页含综合评语、雷达图与分享文案。
- 数据层：questions / outcomes / comments 三份 JSON 作为核心配置。
- 卡牌系统：答题解锁卡牌、卡牌墙选择 3 张、结果页回声拼句展示。

# 当前进度
- 15 题与 6 维度数据已完成迁移与验证。
- 结果判定与评语规则已对齐新维度。
- 卡牌墙、选卡、回声拼句与结果页展示已实现。
- 雷达图标签已做动态留白处理，减少遮挡。

# 现状与情况
- 运行方式：本地静态服务可直接预览，无构建步骤。
- 交互流程：开始页 -> 答题 -> 结果页；卡牌墙可从开始/结果页进入。
- 数据依赖：JSON 文件需与题库、权重、规则一致，否则会影响结果分配。
- 卡牌回声：当前为三张卡牌标题拼接成一句展示。

# 可完善点（按优先级）
## P1 - 体验稳定性
- 卡牌回声在空标题或特殊标点时可进一步容错与清洗。
- 分享图未包含“卡牌回声”信息，可考虑加入。
- 数据一致性可增加校验脚本，避免规则与题库不匹配。

## P2 - 视觉与可读性
- 雷达图在极小屏仍可能拥挤，可加字号自适应或自动换行。
- 结果页可增加“主导维度”高亮模块，提升信息抓取效率。

## P3 - 交互与玩法
- 选满 3 张后支持替换机制（自动替换最早选择或弹窗确认）。
- 结果页可展示“本次引用卡牌”缩略预览。
- 记录最近一次结果，允许结果页再次打开。
